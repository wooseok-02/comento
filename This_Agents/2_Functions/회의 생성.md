# core/meeting/meeting_pipeline.py

## generate_meeting_note_preview
입력:
- meeting_title
- meeting_purpose
- meeting_description
- uploaded_audio_file
- owner_type

처리:
- 입력값을 검증한다.
- 업로드된 음성 파일을 임시 파일로 저장한다.
- transcribe_audio 함수를 호출해 transcript를 생성한다.
- generate_meeting_note 함수를 호출해 회의록 markdown을 생성한다.
- 처리 완료 후 임시 음성 파일을 삭제한다.
- UI에서 사용할 회의록 미리보기 결과를 반환한다.

출력:
- success
- message
- meeting_note

## validate_meeting_input
입력:
- meeting_title
- meeting_purpose
- meeting_description
- uploaded_audio_file
- owner_type

처리:
- 회의 제목 누락 여부를 확인한다.
- 회의 목적 누락 여부를 확인한다.
- 회의 설명 누락 여부를 확인한다.
- 음성 파일 누락 여부를 확인한다.
- 저장 위치 값이 personal 또는 teams인지 확인한다.
- 파일 확장자가 mp3, wav, m4a 중 하나인지 확인한다.

출력:
- success
- message

## save_uploaded_audio_to_temp
입력:
- uploaded_audio_file

처리:
- 업로드된 음성 파일을 임시 디렉토리에 저장한다.
- STT API가 읽을 수 있는 파일 경로를 만든다.

출력:
- temp_audio_path

## transcribe_audio
입력:
- temp_audio_path

처리:
- OpenAI STT API를 호출한다.
- 음성 파일을 텍스트 transcript로 변환한다.

출력:
- transcript

## generate_meeting_note
입력:
- meeting_title
- meeting_purpose
- meeting_description
- transcript

처리:
- 회의 정보와 transcript를 LLM 프롬프트에 전달한다.
- 회의록 기본 구조에 맞는 markdown 회의록을 생성한다.

출력:
- meeting_note_content

## delete_temp_audio_file
입력:
- temp_audio_path

처리:
- STT 처리에 사용한 임시 음성 파일을 삭제한다.

출력:
- 없음

# core/meeting/meeting_storage.py

## save_generated_meeting_note
입력:
- meeting_note

처리:
- 사용자가 수정한 회의록 markdown을 파일로 저장한다.
- 자료관리 metadata를 생성한다.
- Supabase resources table에 회의록 자료로 등록한다.

출력:
- success
- message
- file_name
- file_path
- resource

## create_meeting_note_file_name
입력:
- meeting_title

처리:
- 회의 제목을 파일명으로 사용할 수 있게 정리한다.
- timestamp를 붙인다.
- .md 확장자를 붙인다.

출력:
- file_name

## build_meeting_note_markdown
입력:
- meeting_note

처리:
- 회의 제목, 목적, 설명, 회의록 본문을 markdown 형식으로 조합한다.
- transcript 원문은 기본 저장 대상에 포함하지 않는다.

출력:
- markdown_content

## write_meeting_note_file
입력:
- file_name
- markdown_content

처리:
- Supabase Storage generated-files/meeting_notes 경로에 저장한다.
- markdown 파일을 저장한다.

출력:
- file_name
- file_path

## register_meeting_note_resource
입력:
- meeting_note
- file_name
- file_path

처리:
- resource_type을 meeting_note로 설정한다.
- source_type을 generated_meeting_note로 설정한다.
- owner_type은 사용자가 선택한 저장 위치를 사용한다.
- 자료관리 metadata에 등록한다.

출력:
- resource

# UI/pages/meeting_create.py

## render_meeting_create_page
입력:
- 없음

처리:
- 회의 제목, 목적, 설명 입력 UI를 표시한다.
- 음성 파일 업로드 UI를 표시한다.
- 저장 위치 선택 UI를 표시한다.
- 회의록 생성 버튼을 표시한다.
- 생성 단계별 상태를 표시한다.
- 생성된 회의록 markdown 편집 영역을 표시한다.
- transcript 원문 보기 영역을 표시한다.
- 회의록 저장 버튼을 표시한다.

출력:
- 화면 렌더링
