1. UI 입력 수집
위치: UI/pages/report_create.py
입력값:title
topic
purpose
owner_type


2. UI 입력 검증
제목, 주제, 목적이 비어 있으면 생성 중단
저장 위치가 선택되지 않으면 생성 중단

3. 보고서 생성 요청
UI가 core의 보고서 생성 함수 호출
예:generate_report_preview(title, topic, purpose, owner_type)


4. LangGraph 실행 시작
위치: core/report/report_graph.py
초기 state:title
topic
purpose
owner_type


5. 내부 RAG 검색
Chroma Cloud vector store를 로드
topic + purpose를 query로 사용
관련 문서 검색
결과를 state에 저장
저장값:internal_documents
internal_sources


6. 웹 검색
topic + purpose를 query로 사용
웹 검색 결과 수집
결과를 state에 저장
저장값:web_results
web_sources


7. 참고자료 통합
내부 문서 결과와 웹 검색 결과를 보고서 생성용 context로 정리
저장값:merged_context
sources


8. 보고서 생성
LLM 호출
title, topic, purpose, merged_context를 기반으로 markdown 보고서 생성
저장값:report_content


9. 생성 결과 반환
LangGraph 실행 결과 state를 UI로 반환
UI는 아래 내용을 session_state에 저장title
content
sources
owner_type


10. UI 미리보기 표시
화면에 보고서 본문 표시
출처 목록 표시
보고서 저장 버튼 표시

11. 사용자가 저장 버튼 클릭UI가 core 저장 함수 호출
예:save_generated_report(report_preview)


12. markdown 파일 저장위치: Supabase Storage generated-files/reports/
파일명:보고서 제목 기반 + timestamp

13. 저장 결과:file_path
file_name


14. 자료관리 metadata 등록위치: Supabase resources table
등록값:owner_type
resource_type = "report"
source_type = "generated_report"
title
file_name
file_path


15. 저장 완료 응답UI에 저장 성공 메시지 표시
이후 자료관리에서 다운로드 가능
