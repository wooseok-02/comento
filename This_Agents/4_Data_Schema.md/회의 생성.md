# meeting_note preview

회의록 생성 후 UI 미리보기와 편집에 사용하는 데이터 구조이다.

```json
{
  "meeting_title": "회의 제목",
  "meeting_purpose": "회의 목적",
  "meeting_description": "회의 설명",
  "owner_type": "personal",
  "transcript": "STT 변환 원문",
  "content": "AI가 생성한 회의록 markdown",
  "created_at": "2026-07-12T10:00:00"
}
```

## 필드 설명

### meeting_title
회의 제목이다.

### meeting_purpose
회의 목적이다.

### meeting_description
회의 배경, 참석자, 주요 안건 등 사용자가 입력한 설명이다.

### owner_type
회의록 저장 위치이다.

허용값:
- personal
- teams

### transcript
STT로 변환된 원문이다.
v1에서는 파일로 저장하지 않고 화면의 원문 보기 영역에서만 사용한다.

### content
AI가 생성한 회의록 markdown 본문이다.
사용자가 편집한 뒤 저장되는 최종 대상이다.

### created_at
회의록 생성 시각이다.

# resource metadata

회의록 저장 시 store/resources/resources.json에 등록되는 자료관리 metadata이다.

```json
{
  "resource_id": "uuid",
  "owner_type": "personal",
  "resource_type": "meeting_note",
  "source_type": "generated_meeting_note",
  "title": "회의 제목",
  "file_name": "회의제목_20260712_100000.md",
  "file_path": "store/meeting_notes/회의제목_20260712_100000.md",
  "source_document_id": null,
  "created_at": "2026-07-12T10:00:00"
}
```

## owner_type
자료 저장 범위이다.

허용값:
- personal
- teams

## resource_type
자료 유형이다.

회의록 생성 기능에서는 다음 값으로 고정한다.
- meeting_note

## source_type
자료 생성 출처이다.

회의록 생성 기능에서는 다음 값으로 고정한다.
- generated_meeting_note

## title
회의 제목이다.

## file_name
저장된 markdown 파일명이다.

## file_path
저장된 markdown 파일 경로이다.

## source_document_id
공통 문서에서 파생된 자료가 아니므로 null로 저장한다.

## created_at
자료 등록 시각이다.

# 저장 파일

회의록 markdown 파일은 store/meeting_notes/에 저장한다.

파일 내용에는 다음 정보를 포함한다.
- 회의 제목
- 회의 목적
- 회의 설명
- 사용자가 편집한 회의록 markdown 본문
- 생성일

transcript 원문은 v1에서 저장 파일에 포함하지 않는다.
