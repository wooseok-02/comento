# 설명 : 사용자가 요청했을 때, 시스템 내부에서 어떤 순서로 처리되는지 정한다.

## 흐름 3가지
1. 문서 업로드 및 벡터화 처리 흐름
2. 문서 조회 처리 흐름
3. 활성/비활성 상태 변경 처리 흐름

### 1. 문서 업로드 및 벡터화 처리 흐름
1. 문서 업로드 버튼
2. 문서 형태 파악
2-1. pdf,md,txt면 다음 단계 진행 / 아닐 시 에러 처리 : "올바르지 않은 문서 형식" -> 문서 업로드 취소 상태
3. 문서 중복 확인 -> 파일 해시를 통해서 확인. 중복 없으면 다음 단계 진행 / 아닐 시 에러 처리 : "이미 등록된 문서입니다." -> 문서 업로드 취소
4. 문서 원문 저장 (상태 : 처리중)
5. 문서 메타데이터 저장
6. 텍스트 추출
7. 청킹, 임베딩
8. 벡터 저장
9. 상태 업데이트(상태 : 완료)


### 2. 문서 조회 처리 흐름
1. 특정 카테고리 버튼
2. metadata에서 해당 카테고리에 해당하는 문서 로드
2-1. 업로드 날짜 기준 최신순 정렬

### 3. 활성/비활성 상태 변경 처리 흐름
1. 문서 조회 처리후 활성 상태 버튼 클릭
2. 해당 문서의 metadata 상태 변경

### 4. 벡터 DB 업데이트 처리 흐름
1. RAG문서 업데이트 버튼
2. metadata에서 is active 확인
3. 처리 완료 상태 + 활성 상태 문서만 대상으로 재생성
