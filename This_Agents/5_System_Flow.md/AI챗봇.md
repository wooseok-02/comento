# 설명 : 사용자가 AI 챗봇을 사용할 때, 시스템 내부에서 어떤 순서로 처리되는지 정한다.

## 흐름 2가지
1. 새 대화방 생성 및 질의응답 처리 흐름
2. 기존 대화방 조회 및 이어서 질의응답 처리 흐름

---

## 1. 새 대화방 생성 및 질의응답 처리 흐름

1. 사용자가 챗봇 페이지에 접근한다.
2. 사용자가 새 대화방 생성 버튼을 클릭한다.
3. 사용자가 대화방 제목을 입력한다.
4. 시스템은 새 `chat_room_id`를 생성한다.
5. 시스템은 대화방 metadata를 저장한다.
   - chat_room_id
   - title
   - created_at
   - updated_at
6. 시스템은 생성된 대화방을 현재 대화방으로 설정한다.
7. 사용자가 자연어 질문을 입력한다.
8. 시스템은 사용자 질문 메시지를 저장한다.
   - message_id
   - chat_room_id
   - role: user
   - content
   - sources: []
   - created_at
9. 시스템은 질문을 분석하고 답변에 필요한 정보를 검색한다.
   - RAG 벡터 DB 검색
   - 웹 검색
10. 시스템은 검색된 문서 또는 웹 정보를 기반으로 답변을 생성한다.
11. 시스템은 AI 답변 메시지를 저장한다.
   - message_id
   - chat_room_id
   - role: assistant
   - content
   - sources
   - created_at
12. 시스템은 대화방의 `updated_at`을 최신 메시지 시간으로 갱신한다.
13. 시스템은 화면에 사용자 질문과 AI 답변을 표시한다.
14. 시스템은 AI 답변 아래에 출처를 접기/펼치기 형태로 표시한다.
15. 사용자는 출처의 문서 다운로드 또는 웹 링크 이동을 수행할 수 있다.
16. 사용자가 추가 질문을 입력하면 8번부터 반복한다.

---

## 2. 기존 대화방 조회 및 이어서 질의응답 처리 흐름

1. 사용자가 챗봇 페이지에 접근한다.
2. 시스템은 저장된 대화방 목록을 불러온다.
3. 시스템은 대화방 목록을 `updated_at` 기준 최신순으로 정렬한다.
4. 시스템은 사이드바에 대화방 제목 목록을 표시한다.
5. 사용자가 특정 대화방을 선택한다.
6. 시스템은 선택한 대화방의 `chat_room_id`를 현재 대화방으로 설정한다.
7. 시스템은 해당 `chat_room_id`에 속한 메시지 목록을 불러온다.
8. 시스템은 메시지를 `created_at` 기준 시간순으로 정렬한다.
9. 시스템은 기존 사용자 질문과 AI 답변을 화면에 표시한다.
10. AI 답변 메시지에 `sources`가 있으면 출처 영역을 함께 표시한다.
11. 사용자가 이어서 자연어 질문을 입력한다.
12. 시스템은 사용자 질문 메시지를 저장한다.
    - message_id
    - chat_room_id
    - role: user
    - content
    - sources: []
    - created_at
13. 시스템은 질문을 분석하고 답변에 필요한 정보를 검색한다.
    - RAG 벡터 DB 검색
    - 웹 검색
14. 시스템은 검색된 문서 또는 웹 정보를 기반으로 답변을 생성한다.
15. 시스템은 AI 답변 메시지를 저장한다.
    - message_id
    - chat_room_id
    - role: assistant
    - content
    - sources
    - created_at
16. 시스템은 대화방의 `updated_at`을 최신 메시지 시간으로 갱신한다.
17. 시스템은 새 질문과 답변을 기존 대화 화면 아래에 이어서 표시한다.
