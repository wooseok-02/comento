# 회의 생성 시스템 흐름

## 전체 흐름

```text
UI 입력
→ 입력값 검증
→ 음성 파일 임시 저장
→ STT 변환
→ LLM 회의록 생성
→ 임시 음성 파일 삭제
→ 회의록 미리보기 및 편집
→ 사용자 저장
→ markdown 파일 저장
→ 자료관리 metadata 등록
```

## 1. UI 입력 수집

위치:
- UI/pages/meeting_create.py

입력값:
- meeting_title
- meeting_purpose
- meeting_description
- uploaded_audio_file
- owner_type

## 2. 입력값 검증

검증 항목:
- 회의 제목이 비어 있는지 확인
- 회의 목적이 비어 있는지 확인
- 회의 설명이 비어 있는지 확인
- 음성 파일이 업로드되었는지 확인
- 저장 위치가 personal 또는 teams인지 확인
- 파일 확장자가 mp3, wav, m4a인지 확인

검증 실패:
- 이후 처리를 진행하지 않는다.
- UI에 안내 메시지를 표시한다.

## 3. 음성 파일 임시 저장

위치:
- core/meeting/meeting_pipeline.py

처리:
- 업로드된 음성 파일을 임시 파일로 저장한다.
- STT API가 읽을 수 있는 파일 경로를 생성한다.
- 원본 음성 파일은 영구 저장하지 않는다.

## 4. STT 변환

위치:
- core/meeting/meeting_pipeline.py

처리:
- OpenAI STT API를 호출한다.
- 임시 음성 파일을 transcript 텍스트로 변환한다.

실패 시:
- 회의록 생성을 중단한다.
- 임시 음성 파일을 삭제한다.
- UI에 오류 메시지를 반환한다.

## 5. LLM 회의록 생성

위치:
- core/meeting/meeting_pipeline.py

입력:
- meeting_title
- meeting_purpose
- meeting_description
- transcript

처리:
- 회의 정보를 LLM 프롬프트에 넣는다.
- transcript를 기반으로 회의록 markdown을 생성한다.

회의록 구조:
- 회의 요약
- 주요 논의 내용
- 결정 사항
- 액션 아이템
- 후속 확인 사항

## 6. 임시 파일 삭제

처리:
- STT 처리가 완료되면 임시 음성 파일을 삭제한다.
- STT 또는 LLM 단계에서 실패하더라도 임시 파일 삭제를 시도한다.

## 7. 미리보기 및 편집

위치:
- UI/pages/meeting_create.py

처리:
- 생성된 회의록 markdown을 text area에 표시한다.
- 사용자가 회의록 내용을 직접 수정할 수 있게 한다.
- transcript는 원문 보기 expander에 표시한다.

저장 대상:
- 사용자가 최종 수정한 markdown 본문

## 8. 회의록 저장

위치:
- core/meeting/meeting_storage.py

처리:
- 사용자가 저장 버튼을 누르면 실행한다.
- markdown 파일명을 생성한다.
- store/meeting_notes/에 파일을 저장한다.
- 자료관리 metadata를 생성한다.
- store/resources/resources.json에 등록한다.

## 9. 자료관리 등록

등록값:
- owner_type: personal 또는 teams
- resource_type: meeting_note
- source_type: generated_meeting_note
- title: 회의 제목
- file_name: 저장된 markdown 파일명
- file_path: 저장된 markdown 파일 경로
- source_document_id: null

## 10. 자료관리 조회

저장 완료 후 사용자는 자료관리 화면에서 회의록을 확인할 수 있다.

- 개인 저장: 자료 관리 > 개인 > 회의록
- Teams 저장: 자료 관리 > Teams > 회의록
