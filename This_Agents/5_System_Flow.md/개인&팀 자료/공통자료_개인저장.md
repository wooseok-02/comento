1. 사용자가 공통 자료 조회 페이지에 접근한다.
2. 사용자가 카테고리를 선택한다.
3. 시스템은 Supabase documents table에서 해당 카테고리의 공통 문서 목록을 조회한다.
4. 사용자가 특정 문서의 “내 자료에 저장” 버튼을 클릭한다.
5. 시스템은 해당 문서의 document_id를 확인한다.
6. 시스템은 resources table에서 같은 source_document_id가 personal 자료에 이미 있는지 확인한다.
7. 시스템은 개인 자료 metadata를 생성한다.
   - resource_id
   - owner_type: personal
   - resource_type: file
   - source_type: common_document
   - title
   - file_name
   - file_path
   - storage_bucket
   - storage_path
   - source_document_id
   - created_at
8. 시스템은 Supabase resources table에 개인 자료 metadata를 저장한다.
9. 시스템은 사용자에게 “내 자료에 저장되었습니다.” 메시지를 표시한다.

# 예외처리
이미 같은 source_document_id가 personal 자료에 있으면 중복 저장하지 않는다.
사용자에게 “이미 내 자료에 저장된 문서입니다.” 메시지를 표시한다.
