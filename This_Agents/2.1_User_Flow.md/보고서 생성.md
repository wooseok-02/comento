# 사용자 동작 흐름

1. 사용자가 사이드바에서 업무 자동화 > 보고서 생성을 클릭한다.

2. 보고서 생성 화면이 열린다.

3. 사용자는 보고서 생성에 필요한 정보를 입력한다.
보고서 제목
보고서 주제
보고서 목적
저장 위치: 개인 / Teams

4. 사용자는 보고서 생성 버튼을 클릭한다.

5. 시스템은 입력값을 검증한다.
보고서 제목이 비어 있으면 안내 메시지를 보여준다.
보고서 주제가 비어 있으면 안내 메시지를 보여준다.
보고서 목적이 비어 있으면 안내 메시지를 보여준다.

6. 입력값이 정상이라면 AI 보고서 생성이 시작된다.

7. 시스템은 내부 RAG 문서를 검색한다.
보고서 주제와 목적을 기반으로 관련 문서를 찾는다.
관련 문서가 있으면 보고서 참고자료로 사용한다.

8. 시스템은 웹 검색을 실행한다.
보고서 주제와 목적을 기반으로 최신 외부 정보를 찾는다.
검색 결과가 있으면 보고서 참고자료로 사용한다.

9. 시스템은 내부 RAG 결과와 웹 검색 결과를 함께 사용해 보고서를 생성한다.

10. 생성이 완료되면 화면에 보고서 미리보기를 표시한다.
보고서 제목
보고서 본문
참고 출처

11. 사용자는 생성된 보고서를 확인한다.

12. 사용자가 결과가 마음에 들지 않으면 입력값을 수정한 뒤 다시 보고서 생성을 누를 수 있다.

13. 사용자가 결과를 저장하고 싶으면 보고서 저장 버튼을 클릭한다.

14. 시스템은 보고서를 markdown 파일로 저장한다.
저장 위치: store/reports/

15. 시스템은 자료관리 metadata에 보고서를 등록한다.개인을 선택한 경우: 개인 자료 > 보고서
Teams를 선택한 경우: Teams 자료 > 보고서

16. 저장이 완료되면 사용자에게 저장 완료 메시지를 보여준다.

17. 사용자는 자료관리 페이지로 이동해서 저장된 보고서를 조회하거나 다운로드할 수 있다.

# 예외 흐름
내부 RAG 문서가 없어도 웹 검색 결과만으로 보고서를 생성한다.
웹 검색 결과가 없어도 내부 RAG 문서만으로 보고서를 생성한다.
둘 다 없으면 AI는 “참고자료가 부족하다”는 전제를 두고 일반적인 초안을 생성하되, 출처가 부족하다는 안내를 함께 보여준다.
AI 생성 중 오류가 발생하면 보고서는 저장하지 않고 오류 메시지를 보여준다.
사용자가 저장 버튼을 누르지 않으면 파일과 자료관리 metadata에는 저장하지 않는다.
