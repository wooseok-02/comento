# 회의 생성 사용자 흐름

1. 사용자가 사이드바에서 업무 자동화 메뉴를 연다.

2. 사용자가 회의 생성 메뉴를 클릭한다.

3. 회의 생성 화면이 열린다.

4. 사용자는 회의 제목을 입력한다.

5. 사용자는 회의 목적을 입력한다.

6. 사용자는 회의 설명을 입력한다.

7. 사용자는 회의 음성 파일을 업로드한다.
   - 지원 형식: mp3, wav, m4a

8. 사용자는 저장 위치를 선택한다.
   - 개인
   - Teams

9. 사용자는 회의록 생성 버튼을 클릭한다.

10. 시스템은 입력값을 검증한다.
    - 제목이 없으면 안내 메시지를 표시한다.
    - 목적이 없으면 안내 메시지를 표시한다.
    - 설명이 없으면 안내 메시지를 표시한다.
    - 음성 파일이 없으면 안내 메시지를 표시한다.
    - 지원하지 않는 파일 형식이면 안내 메시지를 표시한다.

11. 입력값이 정상이라면 시스템은 단계별 진행 상태를 표시한다.
    - 입력값 검증 중
    - 음성 텍스트 변환 중
    - 회의록 생성 중
    - 임시 파일 정리 중
    - 미리보기 준비 중

12. 시스템은 업로드된 음성 파일을 임시 파일로 저장한다.

13. 시스템은 OpenAI STT API를 사용해 음성을 transcript로 변환한다.

14. 시스템은 회의 제목, 목적, 설명, transcript를 LLM에 전달한다.

15. LLM은 회의록 기본 구조에 맞춰 markdown 회의록을 생성한다.

16. 시스템은 임시 음성 파일을 삭제한다.

17. 시스템은 생성된 회의록을 편집 가능한 markdown 영역에 표시한다.

18. 시스템은 transcript 원문을 원문 보기 영역에 표시한다.
    - 원문은 기본적으로 접힌 상태로 제공한다.

19. 사용자는 생성된 회의록 markdown을 직접 수정한다.

20. 사용자는 회의록 저장 버튼을 클릭한다.

21. 시스템은 사용자가 수정한 최종 markdown 내용을 저장한다.

22. 시스템은 회의록 파일을 store/meeting_notes/에 저장한다.

23. 시스템은 자료관리 metadata에 회의록을 등록한다.
    - 개인 선택 시 개인 자료의 회의록으로 등록
    - Teams 선택 시 Teams 자료의 회의록으로 등록

24. 시스템은 저장 완료 메시지를 표시한다.

25. 사용자는 자료관리 화면에서 저장된 회의록을 조회하거나 다운로드할 수 있다.

# 예외 흐름

## 입력값 누락
필수 입력값이 누락되면 회의록 생성을 시작하지 않고 안내 메시지를 표시한다.

## 파일 형식 오류
mp3, wav, m4a가 아닌 파일이 업로드되면 회의록 생성을 시작하지 않고 안내 메시지를 표시한다.

## STT 실패
음성 텍스트 변환에 실패하면 회의록 생성을 중단하고 오류 메시지를 표시한다.
임시 음성 파일은 삭제한다.

## 회의록 생성 실패
LLM 회의록 생성에 실패하면 저장 버튼을 표시하지 않고 오류 메시지를 표시한다.

## 저장 실패
회의록 파일 저장 또는 자료관리 등록에 실패하면 저장 실패 메시지를 표시한다.

## 사용자가 저장하지 않음
사용자가 저장 버튼을 누르지 않으면 회의록 파일과 자료관리 metadata는 생성하지 않는다.
