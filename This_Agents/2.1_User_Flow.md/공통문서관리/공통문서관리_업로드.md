# 설명 : 관리자가 문서를 업로드할 때의 총 flow를 정리한 문서. 

# 전체 흐름 관리

관리자가 문서 관리 화면에 접근
↓
문서 업로드 영역에서 파일 선택
- 이 시점에는 파일의 분류를 선택하지 않음
↓
시스템이 파일 형태 자동 판단 및 표시
- PDF
- TXT
- MD
↓
지원 가능한 파일이면 다음 단계 진행
↓
관리자가 문서 카테고리 선택
- 규정
- 내규
- 법령
- 판례
- 보도자료
↓
관리자가 업로드 버튼 클릭
↓
현재 문서 처리 과정 영역에 파일 제목과 처리 상태 표시
(예시)
{시스템이 처리 단계를 순차적으로 표시
- 파일 해시 생성 중
- 중복 검사 중
- 원본 파일 저장 중
- 텍스트 추출 중
- 문서 청킹 중
- 임베딩 생성 중
- 벡터 DB 저장 중
↓
처리 결과 표시
- 완료
- 실패
- 중복}
↓
시스템이 파일 내용 해시 생성
↓
시스템이 해시를 통해 중복 여부 판단
↓
중복이 아닌 경우
↓
파일 업로드 완료
↓
화면에 파일 제목 생성
- 파일명 기반 문서 제목
↓
화면에 현재 파일 처리 상태 표시
- 처리중
- 완료
- 실패
- 중복
↓
시스템이 파일 벡터화 수행
- 텍스트 추출
- 청킹
- 임베딩
↓
시스템이 벡터 DB 저장
↓
시스템이 처리 상태 업데이트
↓
관리자가 완료 메시지 확인

# 중복 흐름 관리

시스템이 해시를 통해 중복 여부 판단
↓
동일한 해시가 이미 존재
↓
시스템이 벡터화를 진행하지 않음
↓
화면에 파일 제목과 중복 상태 표시
↓
관리자가 중복 메시지 확인

# 실패 흐름 관리

파일 형태 판단 실패
또는 텍스트 추출 실패
또는 벡터화 실패
또는 DB 저장 실패
↓
시스템이 처리 상태를 실패로 업데이트
↓
관리자가 실패 메시지 확인
