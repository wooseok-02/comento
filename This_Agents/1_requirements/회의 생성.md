# 기능명
회의 생성

# 목적
사용자가 업로드한 회의 음성 파일을 AI가 텍스트로 변환하고, 변환된 내용을 회의록 형식으로 정리한다.
생성된 회의록은 사용자가 화면에서 직접 수정한 뒤 개인 또는 Teams 자료관리의 회의록으로 저장할 수 있다.

# 사용자 입력
- 회의 제목
- 회의 목적
- 회의 설명
- 회의 음성 파일
- 저장 위치: 개인 / Teams

# 지원 파일 형식
- mp3
- wav
- m4a

# 원본 음성 파일 정책
원본 음성 파일은 회의록 생성 과정에서만 임시로 사용한다.
STT 처리가 완료되면 임시 음성 파일은 삭제한다.
원본 음성 파일은 자료관리 또는 별도 저장소에 영구 저장하지 않는다.

# transcript 정책
STT로 생성된 원문 transcript는 별도 파일로 저장하지 않는다.
화면에서는 생성 완료 후 원문 보기 영역에서만 확인할 수 있다.
최종 저장 대상은 사용자가 수정한 회의록 markdown 본문이다.

# 처리 방식
1. 사용자가 회의 제목, 목적, 설명을 입력한다.
2. 사용자가 mp3, wav, m4a 형식의 음성 파일을 업로드한다.
3. 사용자가 저장 위치를 개인 또는 Teams 중 선택한다.
4. 사용자가 회의록 생성 버튼을 클릭한다.
5. 시스템은 입력값과 파일 형식을 검증한다.
6. 시스템은 업로드된 음성 파일을 임시 파일로 저장한다.
7. OpenAI STT API를 사용해 음성을 텍스트로 변환한다.
8. 변환된 transcript와 사용자가 입력한 회의 정보를 LLM에 전달한다.
9. LLM은 회의록 기본 구조에 맞게 markdown 회의록을 생성한다.
10. 시스템은 임시 음성 파일을 삭제한다.
11. 생성된 회의록과 transcript 원문을 화면에 표시한다.
12. 사용자는 생성된 회의록 markdown을 직접 수정할 수 있다.
13. 사용자가 저장 버튼을 누르면 수정된 회의록을 markdown 파일로 저장한다.
14. 저장된 회의록은 자료관리 metadata에 등록된다.

# 회의록 기본 구조
- 회의 요약
- 주요 논의 내용
- 결정 사항
- 액션 아이템
- 후속 확인 사항

# 저장 방식
- 회의록 파일 저장 위치: store/meeting_notes/
- 파일 형식: .md
- 자료관리 등록 위치: store/resources/resources.json
- owner_type: personal 또는 teams
- resource_type: meeting_note
- source_type: generated_meeting_note

# 단계별 상태 표시
회의록 생성 과정은 사용자가 진행 상태를 알 수 있도록 단계별로 표시한다.
- 입력값 검증
- 음성 텍스트 변환
- 회의록 생성
- 임시 파일 정리
- 미리보기 준비
- 저장 처리

# 긴 회의 파일 처리
v1에서는 긴 음성 파일 자동 분할 처리를 제외한다.
긴 회의 녹음은 사용자가 짧게 분할한 파일을 업로드하는 것을 권장한다.
향후 확장으로 오디오 chunk 분할 STT 처리를 추가할 수 있다.
