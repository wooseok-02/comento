# 목적
Hugging Face Spaces에 Streamlit 앱을 배포하고, AI가 생성한 산출물은 Supabase에 저장한다.

# 배포 플랫폼
- Hugging Face Spaces
- SDK: Streamlit
- 진입점: app.py

# 클라우드 저장소
- Supabase Database
- Supabase Storage

# 적용 범위
이번 배포 단계에서는 모든 저장소를 클라우드로 이전하지 않는다.
시간 대비 학습 효과와 구현 가능성을 고려해 생성 산출물 저장 흐름만 Supabase로 전환한다.

Supabase로 저장하는 대상:
- 보고서 markdown 파일
- 회의록 markdown 파일
- 자료관리 metadata

로컬 저장을 유지하는 대상:
- 공통 문서 업로드 파일
- 공통 문서 metadata
- FAISS vector store
- AI 챗봇 대화방
- AI 챗봇 메시지
- RAG 업데이트 산출물

# Supabase Storage 요구사항
bucket 이름:
- generated-files

저장 경로:
- reports/{file_name}
- meeting_notes/{file_name}

# Supabase Database 요구사항
table 이름:
- resources

역할:
- 개인/Teams 자료관리 목록 조회
- 보고서와 회의록 metadata 저장
- Supabase Storage 파일 경로 저장

# 환경변수
로컬과 Hugging Face Spaces에서 아래 환경변수를 사용한다.

- OPENAI_API_KEY
- TAVILY_API_KEY
- SUPABASE_URL
- SUPABASE_SERVICE_ROLE_KEY
- SUPABASE_STORAGE_BUCKET

SUPABASE_STORAGE_BUCKET은 선택값이며, 없으면 generated-files를 기본값으로 사용한다.

# 보안 원칙
- API key는 코드에 직접 작성하지 않는다.
- 로컬에서는 .env를 사용한다.
- Hugging Face Spaces에서는 Repository secrets에 등록한다.
- SUPABASE_SERVICE_ROLE_KEY는 외부에 노출하지 않는다.

# 사용자 관점 요구사항
- 사용자는 배포된 URL에서 앱에 접속할 수 있다.
- 사용자는 보고서를 생성하고 저장할 수 있다.
- 사용자는 회의록을 생성하고 저장할 수 있다.
- 저장된 보고서와 회의록은 자료관리 화면에서 조회할 수 있다.
- 저장된 산출물은 다운로드할 수 있다.

# 제한사항
- 현재 버전은 사용자 인증을 제공하지 않는다.
- Supabase RLS 정책은 데모 편의를 위해 단순화한다.
- 공통 문서와 RAG vector store는 여전히 로컬 파일 기반이다.
- Space 재시작 시 로컬 store 기반 데이터는 유지되지 않을 수 있다.
- Supabase로 저장된 보고서/회의록 산출물은 유지된다.

# 향후 확장
- 공통 문서 파일을 Supabase Storage 또는 S3로 이전
- 문서 metadata를 Supabase DB로 이전
- 채팅방과 메시지를 Supabase DB로 이전
- FAISS를 pgvector, Chroma Cloud, Pinecone 같은 외부 vector DB로 이전
- 사용자 인증과 권한 분리 추가
- Supabase RLS 정책 정교화
