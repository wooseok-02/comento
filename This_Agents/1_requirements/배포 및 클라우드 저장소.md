# 목적
Streamlit Community Cloud에 앱을 배포하고, 배포 환경에서도 문서 업로드, RAG 검색, 챗봇, 자료관리, 보고서, 회의록 기능이 유지되도록 외부 저장소를 사용한다.

# 배포 플랫폼
- Streamlit Community Cloud
- 진입점: app.py
- 의존성 파일: requirements.txt
- Python 버전 파일: runtime.txt

# 외부 저장소
- Supabase Database
- Supabase Storage
- Chroma Cloud

# 저장소 역할

## Supabase Database
서비스 metadata를 저장한다.

저장 대상:
- 공통 문서 metadata
- 개인/Teams 자료 metadata
- AI 챗봇 대화방
- AI 챗봇 메시지
- 보고서/회의록 자료 metadata

## Supabase Storage
원본 파일과 생성 산출물 파일을 저장한다.

저장 대상:
- 업로드 문서 원본 파일
- 생성 보고서 markdown 파일
- 생성 회의록 markdown 파일

## Chroma Cloud
RAG 검색을 위한 벡터 데이터를 저장한다.

저장 대상:
- 문서 chunk
- embedding vector
- chunk metadata

# 적용 범위
이번 배포 단계에서는 로컬 store를 운영 저장소로 사용하지 않는다.

외부 저장소 전환 대상:
- 공통 문서 업로드 파일
- 공통 문서 metadata
- RAG vector store
- AI 챗봇 대화방
- AI 챗봇 메시지
- 자료관리 metadata
- 보고서 markdown 파일
- 회의록 markdown 파일

# 환경변수
로컬에서는 .env에 입력하고, Streamlit Community Cloud에서는 App secrets에 입력한다.

필수:
- OPENAI_API_KEY
- TAVILY_API_KEY
- SUPABASE_URL
- SUPABASE_SERVICE_ROLE_KEY
- SUPABASE_STORAGE_BUCKET
- SUPABASE_DOCUMENT_BUCKET
- CHROMA_API_KEY
- CHROMA_TENANT
- CHROMA_DATABASE
- CHROMA_COLLECTION_NAME

선택:
- OPENAI_EMBEDDING_MODEL

# 기본 이름
- SUPABASE_STORAGE_BUCKET: generated-files
- SUPABASE_DOCUMENT_BUCKET: uploaded-documents
- CHROMA_COLLECTION_NAME: worldvision_documents
- OPENAI_EMBEDDING_MODEL: text-embedding-3-small

# 보안 원칙
- API key는 코드에 직접 작성하지 않는다.
- 로컬에서는 .env를 사용한다.
- Streamlit Community Cloud에서는 App secrets를 사용한다.
- SUPABASE_SERVICE_ROLE_KEY는 외부에 노출하지 않는다.
- Chroma API key도 외부에 노출하지 않는다.

# 사용자 관점 요구사항
- 사용자는 배포 URL에서 문서를 업로드할 수 있다.
- 업로드된 문서는 배포 앱 재시작 후에도 조회할 수 있다.
- 사용자는 RAG 문서 업데이트 버튼으로 Chroma Cloud에 벡터 데이터를 반영할 수 있다.
- AI 챗봇은 Chroma Cloud에서 내부 문서를 검색해 답변한다.
- AI 챗봇 대화방과 메시지는 Supabase에 유지된다.
- 사용자는 공통 문서를 개인 자료에 저장할 수 있다.
- 사용자는 보고서와 회의록을 생성하고 Supabase에 저장할 수 있다.
- 사용자는 자료관리 화면에서 개인/Teams 자료를 조회하고 다운로드할 수 있다.

# 제한사항
- 현재 버전은 사용자 인증을 제공하지 않는다.
- owner_type은 personal/teams로만 구분한다.
- Supabase RLS 정책은 초기 배포 편의를 위해 단순화한다.
- 문서 삭제 기능은 포함하지 않는다.
- Chroma Cloud collection은 RAG 업데이트 시 활성 문서를 기준으로 재생성한다.
