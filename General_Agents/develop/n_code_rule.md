# 주의사항
- 모든 코드를 작성해서 보여주지 말 것. 항상 내가 요구한 것에 해당하는 코드를 제공할 것
- 임의로 내 파일에 코드를 이식하지 말 것. 항상 요청 -> 나의 승인으로 코드를 이식할 것
- 코드 작성시에는 기능당 한 줄의 주석을 달 것
- 코드 작성 후 테스트하지 말 것
- 모든 코드가 완료되면 항상 변경사항을 요약해서 보고할 것
- 프로젝트는 너의 방향 제안 -> 나의 승인으로 진행될 것
- 설계,개발도중 너 고려해야 되는 부분이나 하면 안되는 부분은 명확히 짚어줄 것

# 프로젝트 진행 원칙

- 이 프로젝트의 모든 작업은 바로 코드 작성으로 들어가지 않고, 먼저 설계 흐름을 따른다.
- 작업의 최종 결정권은 항상 사용자에게 있다.
- Codex는 먼저 방향을 제안하고, 사용자가 승인한 경우에만 다음 단계로 진행한다.
- Codex는 사용자의 명시적 승인 없이 파일을 수정하거나 코드를 이식하지 않는다.
- 사용자가 "적용해줘", "진행해", "이대로 해줘"처럼 명확히 승인한 경우에만 실제 파일을 변경한다.

# 개발 전 설계 흐름

설계는 Codex가 미리 정한 범위를 사용자에게 확인받는 방식으로 진행하지 않는다.
Codex는 먼저 기능의 목적, 사용자, 사용 장면, 해결하려는 문제를 사용자와 대화하면서 요구사항을 도출한다.
Codex는 사용자가 말하지 않은 입력값, 화면 요소, 데이터 항목, 처리 방식을 전제로 확정하지 않는다.
Codex가 제안할 수는 있지만, 제안은 항상 "가능한 선택지"로 제시하고 사용자의 판단을 기다린다.
요구사항은 필요성, 사용자 행동, 시스템 결과가 드러나는 문장으로 정리한다.

# 코드 작성 원칙

- 사용자가 요청한 범위에 해당하는 코드만 작성한다.
- 전체 코드를 한 번에 작성하지 않고, 기능 단위로 나누어 제안한다.
- 코드 작성 전에는 어떤 파일을 왜 수정해야 하는지 먼저 설명한다.
- 코드 작성 시 기능당 한 줄의 주석을 포함한다.
- 코드 작성 후 테스트는 실행하지 않는다.
- 코드 작성이 끝나면 변경사항을 요약해서 보고한다.
